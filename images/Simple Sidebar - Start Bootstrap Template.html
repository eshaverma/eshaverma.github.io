SSL Certificate Management App






Ideation and Research Product Development In-Production Code


Bringing Current Web SSL Retail Portal to Mobile What does this solve? Easier purchase and management of SSL certificates by customer Can send customer push messages. These can be security-related (“You have two SHA-1 certificates that need to be replaced”) or marketing-related (“Black Friday Special – EV Certificate for only $599”). Can build other usability enhancements not possible on a web interface (ideas discussed later) Display CSR in QR code format on SSLAssistant. Use Mobile App to take scan code and upload CSR for enrollment Leverage Trust Center Customer portal to manage documents needed by Auth team. Customer Usability/Productivity fix: Notify customer of pending/optional documents to hasten customer authentication and hence cert issuance, and let them upload it through Trust Center portal. Replaces manually emailing documents. Easy to know what was submitted. Last known state? Concept researched by Michael Klieman. Engineering would need to build a lightweight catalog service, to file customer documents with Auth team database User Experience design required for design of single page Functionality can be built into mobile app too. Retail team has existing services to send back customer data to any client Javascript app client was built and packaged as native app. Demonstrated at CuttingEdge 2014 from a technology stack perspective Most code already exists . Needs to be picked up and scoped for a release
